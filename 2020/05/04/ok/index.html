笔记

首先

要完成准备工作

  1. 安装git
  2. 安装node js
  3. 注册github

git 下载地址：
git 下载

安装：
一路next就OK

node 下载地址：
node js 下载

安装：一路next 但是在Costom Setup这一步选一下 Add TO Path 这样就不用配置环境变量了

PS：装完之后不放心的话 就在cmd中输入path康康有没有这个路径 没有的话 俺没办法 自己解决叭

说明：LTS为长期支持版，Current为当前最新版 根据自己喜欢的装 我装的是长期的辣个

打开git 或者cmd （命令行）

输入 git version 查看git版本

输入 node -v 查看node版本

ps：cmd 打开方式：
win+r 然后输入cmd 然后确认

什么 win 是什么？emmmmm 就是那个长的和开始一样的那个键

注册github:
这个如果没有耐心的话 会很难 我当时就是有点急 以为电脑坏掉了 按他的要求弄就OK啦
github 网址：GitHub

然后

打开你的git bash

打开方式 在桌面右击鼠标 然后选择 Git Bash Here

开始表演

First

先在你喜欢的位置建一个文件夹 取一个你喜欢的名字(这就是你博客的文件夹咯)

然后回到我们的Bush

Second

接下来 重头戏要来了！好可怕 快跑

创建hexo

先cd 到那个有着你喜欢的名字的文件夹
bash的命令是：

cd /xx/x/x/x/

举个栗子：cd /d/github/blog/

然后输入安装hexo 命令：

npm i -g hexo

等待安装好以后 输入 hexo -v 查看hexo版本

之后输入：hexo init 初始化 hexo

这个完成后 打开辣个你喜欢的文件夹(其实就是你的博客文件夹) 就可以看到里面多了一堆东西 就说明你成功啦

third

搭桥到github

第一步 登录你的github
创建一个 new repository (就是新储存库啦)

然后 新库的名字叫 yourname.github.io
(不是输入yourname 这个地方是你的github名字嗷)

然后创建就好

之后 再回到我们的万恶的 可爱的Bash

fourth

配置 github

输入： git config --global user.name "yourname"

回车

再输入：git config --global user.email "youremail"

回车

yourxxxx就是字面意思 就是你的 别直接复制粘贴嗷

生成ssh

输入： ssh-keygen -t rsa -C "youremail"
(需要连续按三次回车)

然后我们就去寻找生成的这个鬼玩意

输入： cd ~/.ssh

然后输入：cat id_rsa.pub 就会看到你的密钥

然后点开你已经登录的github 点击你的头像旁边的辣个倒三角

然后点击setting（设置） 然后选择 SSH and GPG keys

点击 new SSH key title 随便填 然后把那个刚刚找到的密钥 粘贴过来 然后点击 add ssh key 就OK啦

然后继续回到Bash 输入：ssh -git@github.com 如果不报错 则说明成功

fifth

打开你的博客文件夹 然后用编译器打开 _config.yml 文件(我用的是vscode)

然后修改文件内容 应该是在文件最后

然后

输入命令：npm i hexo-server(hexo 在3.0以下的不用输入)
