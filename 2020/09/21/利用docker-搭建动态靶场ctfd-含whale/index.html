    部署ctfd

    用赵师傅的插件 （入股不亏）

    (一定要科学上网 不然error到你崩溃)

    然后编辑配置文件使token随机

    token 随机其实就是token=randomme

    看一眼就成 正常情况下本来就是随机的

    然后嘛修改一下 Dockerfile 中的源 之前的那个有问题

    大致需要把tunu改成aliyun
    改好后配置如下

    一定要对照好 正常改三到四处 小f第一次就是因为没改全 一直报错啊报错

    然后安装插件并启动

    正常情况下就可以了

    最后检查是否正确启动镜像

    访问服务器8000端口开始配置

    访问 服务器ip:8000

    看到这样的图

    然后按照提示写完 最后点最下面的那个finish（别点上面的）

    然后就进入了 这个页面：

    点击登录 admin

    然后进入这样的页面

    点击上面的ctfd whale 进入配置页面

    然后如下配置

    Docker API URL： unix://var/run/docker.sock

    Frp API IP： 172.1.0.3

    Frp API Port： 7400

    Frp Http Domain Suffix： None

    Frp Http Port： 80

    Frp Direct IP Address ：自己的域名 或者ip

    Frp Direct Minimum Port :28000

    Frp Direct Maximum Port :28100
    Max Container Count :100

    Max Renewal Times: 5

    Frp config template 这里填写之前 frpc.ini 的内容

    (可以 cat frpc.ini查看)

    Docker Auto Connect Containers： ctfd_frpc_1

    Docker Auto Connect Network ：ctfd_frp_containers

    Docker Dns Setting：可写None或填写外部dns

    Docker Swarm Nodes：linux-1

    Docker Multi-Container Network Subnet：173.0.0.0/16

    Docker Multi-Container Network Subnet New Prefix：24

    Docker Container Timeout：3600

    然后点击update 更新

    这样就完成啦

    最后部署一个题试一试能不能用

    点开challenges 然后点加号

    第一个选dynamic_docker

    剩下的看心情

    Choose Challenge Type：Dynamic docker

    Name：test

    Category：test

    Initial Value：1

    Decay Limit：1

    Minimum Value：1

    Docker Image：

    Frp Redirect Type：Direct

    Frp Redirect Port：80

    Score Type：dynamic score

    然后闲的没事的话可以再注册一个号登陆一下

    然后就OK辣 你也可以自己编个flag提交一下 很快乐的
